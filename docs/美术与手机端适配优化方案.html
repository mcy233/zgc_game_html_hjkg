潜空间矿工：美术优化与 9:20 手机端适配方案

当前版本已经解决了不少“一屏显示”和高刷速度问题，但核心玩法仍建立在 800×600（4:3）舞台上。对于约 9:20 的手机竖屏，单纯等比缩放会让游戏只占屏幕中间一小块。 下一轮优化应从“缩放一个 4:3 画面”升级为“按设备比例重排游戏舞台”。

目标设备：手机竖屏 9:20 保留桌面 4:3 体验 避免画面拉伸 美术与布局同步升级

一、当前问题判断

4:3 舞台在 9:20 竖屏上天然偏小

以 360×800 手机为例，如果 800×600 舞台按宽度等比缩放，最终画面约为 360×270，只占屏幕高度三分之一左右。公式是正确的，但视觉结果不好。

继续放大 4:3 会产生裁切

如果强行按高度铺满，4:3 舞台需要约 1066×800，手机宽度放不下， 会横向裁切，影响钩子、资源、HUD 和操作区域。

画面内容偏“工程原型”

现有资源大量依赖 emoji、文字标签和基础色块，主题表达清晰，但在小屏上会显得碎、 小、密，缺少统一的材质、光效、层级和角色辨识度。

手机端需要专门的信息架构

竖屏不是桌面画面的缩小版。它更适合上方状态栏、中间竖向玩法区域、下方操作/任务/说明区的 “长屏游戏”布局。

二、分辨率适配核心方案

建议引入 Responsive Stage Modes：桌面和横屏继续使用 4:3， 手机竖屏切换为 9:20 专用布局。核心原则是：不拉伸内容， 而是按比例重新组织舞台空间。

当前：4:3 等比缩放到手机竖屏

4:3 游戏框
中间小块显示

优点是安全，不裁切；缺点是上下空白巨大，实际可玩区域过小。

建议：9:20 手机竖屏布局

纵向玩法区域
资源与钩子重新布局

利用竖向空间：上方状态，中间玩法，下方任务/操作，视觉上接近手机小游戏。

场景 建议比例 舞台策略 说明
桌面 / 平板横屏 4:3 或 16:9 容器内 4:3 保留当前 800×600 逻辑舞台 保持原版黄金矿工体验，适合鼠标和键盘。
手机横屏 16:9 / 20:9 横向 4:3 舞台居中，HUD 缩放，遮罩层全屏 已有基础可继续沿用，只需加强美术背景填充。
手机竖屏 9:20 新增 portrait 舞台模式，例如 720×1600 玩法区域纵向重排，避免只显示中间小窗口。

三、手机竖屏实现建议

阶段 1：低风险视觉填充

保留 800×600 游戏逻辑，但手机竖屏外层使用 9:20 容器。 4:3 游戏区上移并放大到宽度，底部新增任务、道具、提示面板，上下空白用背景和 UI 填充。

阶段 2：纵向玩法画布

竖屏时使用 720×1280 或 720×1440 的逻辑玩法区。 钩子锚点、物品生成区域、碰撞边界根据模式切换。

阶段 3：完全 9:20 体验

手机竖屏使用完整 720×1600 设计稿比例。 顶部状态栏、中部潜空间、底部操作/补给入口全部按竖屏重新排版。

推荐架构

  • 新增 useViewportMode()：根据 width / height 判断 desktop、landscape、portraitTall。
  • 新增 stageConfig：集中定义逻辑宽高、矿工位置、HUD 区域、物品生成边界、最大钩长。
  • 将 CANVAS_WIDTH、CANVAS_HEIGHT 从常量改为运行时配置，或保持常量并增加 activeStage。
  • 物品坐标生成从绝对像素改为“归一化槽位”：例如 x = slot.x * stage.width，y = slot.y * stage.height。
  • 绘制函数全部读取 stage.width、stage.height，避免 800×600 硬编码扩散。

四、美术优化方向

主题关键词

“潜空间”“AI 博士”“中关村创新核心区”可以做成统一视觉语言： 深色空间背景、数据流网格、霓虹节点、论文卡片、算力芯片、实验文件夹。

  • 主色：深蓝黑、青蓝高光、少量紫色
  • 奖励色：绿色代表积分增长，金色代表稀有成果
  • 危险色：红色用于 DDL、爆炸、失败压力

资源图标替换 emoji

emoji 在不同系统上样式差异大，缩放后容易糊或风格跳跃。建议逐步替换为项目自有 SVG / Canvas sprite。

  • 数据点：发光粒子 / 小型矩阵节点
  • 数据集：叠放数据卡片 / 文件簇
  • 语料库：发光知识库立方体
  • 顶会成果：蓝色钻石 / 奖章
  • 阻碍项：灰色审稿意见石块 / DDL 警示块

角色与钩子

博士小人应从 emoji 升级为一个小型像素/矢量角色：学士帽、实验服、终端屏幕。 钩子可以变成“探索向量”或“检索光束”，和 AI 主题更贴合。

HUD 与补给站

HUD 从黄色木牌升级为“学院终端面板”：半透明玻璃、扫描线、芯片边框。 补给站可以保留木质柜台趣味，但建议加入学院徽章、算力货架和导师气泡。

五、具体实施路线

Step 1

先做视觉安全改造

保留当前玩法逻辑，新增 9:20 外层布局、背景填充、底部信息面板，让手机竖屏不再像“小窗口游戏”。

Step 2

抽象舞台配置

将 CANVAS_WIDTH、CANVAS_HEIGHT、MINER_X、MINER_Y 等常量迁移到 stageConfig，为真正竖屏玩法做准备。

Step 3

重排物品生成与碰撞边界

竖屏时扩大纵向玩法区域，资源分布从“横向矿区”改为“纵深潜空间”，保证钩子路径和资源密度仍然合理。

Step 4

替换美术资产

优先替换博士小人、钩子、资源、阻碍物、HUD 面板。资产格式建议 SVG 优先，复杂特效用 Canvas 粒子补充。

六、我建议优先采用的版本

最稳妥的下一步是做一个 “手机竖屏增强版”： 桌面和横屏继续保留 4:3，手机竖屏使用 9:20 容器，把 4:3 核心玩法区放大并上移， 下方增加任务、补给、操作提示与当前目标。这样风险低，不需要立刻重写碰撞和生成逻辑， 但视觉上能明显改善“中间只有小区域有显示”的问题。

第二阶段再做真正的 纵向玩法画布，让钩子可以在更高的潜空间中探索。 这一步会影响关卡平衡、物品生成、钩子长度、碰撞边界和 UI 锚点，需要测试更多。
