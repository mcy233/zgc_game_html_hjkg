# 潜空间矿工：Q 版科幻美术 AI 提示词方案

本文档用于统一生成后续游戏美术参考图与可切图素材。当前美术目标以 **手机竖屏 9:20** 为主，整体方向为：

> Q 版科幻、AI 博士培养、深蓝潜空间、学院终端 UI、霓虹蓝紫光效、适合 HTML5 / Canvas 小游戏。

## 通用风格提示词

所有提示词都可以追加这一段：

```text
Q版科幻风，深蓝黑色背景，霓虹蓝和科技紫光效，少量金色点缀，圆角玻璃拟态UI，精品手机小游戏，清晰可读，小尺寸也能辨认，适合HTML5 Canvas游戏。如需抠图素材，不要透明背景，统一使用纯洋红色背景（#FF00FF，RGB 255,0,255），主体边缘尽量干净，避免洋红色反光污染主体。不要真实照片，不要复杂文字，不要乱码，不要水印，不要签名，不要写实人物，不要过度杂乱
```

## 抠图背景规范

由于当前生图模型不会稳定输出透明通道，所有本应使用透明背景的素材，统一改为：

```text
纯洋红色背景（#FF00FF，RGB 255,0,255，后续用色键抠成透明）
```

注意事项：

- 洋红背景必须尽量纯净、平整，不要渐变、阴影、纹理。
- 主体不要使用接近洋红的颜色，尤其避免紫红色边缘光、粉色高光、玫红描边。
- 角色、图标、按钮等主体与洋红背景之间要有清晰边界。
- 如果需要发光效果，尽量使用蓝色、青色、金色、绿色，不要使用洋红色光晕。
- `sprite_generate` 工具已下载到本地 `local_tools/sprite_generate`，并已通过 `.git/info/exclude` 本地忽略，不会上传到项目仓库。
- 该工具仓库当前 README 主要说明的是 **黑底软色键** 与视频抠像流程；洋红色背景素材后续可以扩展为洋红色键，或先做颜色转换后走软色键流程。工具参考：[mcy233/sprite_generate](https://github.com/mcy233/sprite_generate)。

## 交互状态规范

凡是按钮、道具槽、商品卡片、弹窗操作区等可交互元素，生成素材时优先要求一组状态图，而不是单张静态图：

- **正常态**：清晰、饱满、可点击，作为默认显示。
- **悬停/触摸高亮态**：边框或背景略微增强，适合电脑鼠标悬停和手机触摸反馈。
- **按下态**：按钮略微下沉、亮度降低或内阴影增强，表达已经按下。
- **选中态**：蓝色或金色描边增强、外发光更明显，适合道具槽、商品卡片、当前选择。
- **禁用态**：低饱和、灰蓝色、透明度降低、发光减弱，表达不可点击或条件不足。
- **已购买/已使用态**：适合补给商品和一次性道具，可带小型对勾、锁定、已激活占位图标，但不要真实文字。

生成时可以要求“横向排列成状态表”或“2行3列状态板”，便于后续切图。
