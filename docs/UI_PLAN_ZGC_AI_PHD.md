# 中关村 AI 博士学院 ×《潜空间矿工》UI 策划（Agent 参考）

> **定位**：本文档以 **Cursor / Agent 实现与对齐设计** 为主，不要求人类通读。人类执行步骤、Figma 协作与 AI 生图提示词见 **`docs/生图与分工清单.md`**。  
> 主题：**北京中关村学院 · 人工智能方向博士生培养**；气质：**活泼、会玩梗、但不低俗**。

---

## 一、产品定位与体验目标

| 维度 | 说明 |
|------|------|
| 核心隐喻 | 「潜空间」= 课题与数据的地下矿层；钩子 = 探索向量 / 实验抓手；积分 = 培养积分 |
| 情绪曲线 | 开局轻松玩梗 → 局中紧张抓资源 → 过关爽感 → 补给站「剁手」幽默 → 长线答辩仪式感 |
| 视觉目标 | 一眼能认出「学院 + AI + 街机矿工」三类元素；避免廉价 Flash 感；**矢量友好**（便于多分辨率与后期换皮） |

---

## 二、品牌与叙事层（文案梗库方向）

以下为**方向示例**，正式文案可由学院宣传口径微调；设计时预留**两行标题 + 一段副文案**位置。

- **世界观梗**：不是在挖矿，是在「挖潜空间」「捞数据集」「钓 Oral」。
- **阶段梗**：博一 = 新生进村；开题 = 盲审巨石；中期 = 咖啡续命；答辩 = 终局 Boss 房。
- **UI 微文案**：按钮不写「确定」可写「开挖」「进站」「去补给站进货」；失败可写「培养线没过，但论文还会再投」等（与现有失败页风格一致即可）。

**禁止**：真实师生姓名、攻击性梗、与学位政策冲突的承诺性表述。

## 三、设计系统（Design Tokens）建议

设计稿与代码应对齐下列 token（Figma Variables / CSS 变量同名）。

### 3.1 色彩

| Token | 用途 | 建议色相 |
|--------|------|----------|
| `--bg-deep` | 全局背景、矿层深处 | 深蓝近黑 `#020617`～`#0f172a` |
| `--surface-hud` | 顶栏、面板底 | 琥珀金渐变（致敬经典矿工条）+ 暗部描边 |
| `--accent-ai` | 强调、链接、科技点缀 | 天青 `#0ea5e9` / 青绿渐变 |
| `--success` | 达标、购买成功 | 翠绿 / 青柠按钮 |
| `--danger` | 失败、TNT、回收通道 | 红 `#ef4444` 系 |
| `--text-primary` | 主文字 | 近白 `#f8fafc` |
| `--text-muted` | 说明文 | `slate-400/500` |

### 3.2 字体

- **数字与 HUD**：选一款**等宽或半宽数字**突出的字体（如 DIN、JetBrains Mono、阿里妈妈数黑体等），培养积分、倒计时、价签统一。
- **标题**：粗黑无衬线（思源黑 Heavy、阿里巴巴普惠体 Heavy）。
- **正文**：常规无衬线，保证移动端 12px 仍可读。

### 3.3 形状与质感

- 外框：**粗描边街机框体**（现有 `border-slate-800` 思路保留），可加分层内阴影模拟压克力面板。
- 圆角：外框大圆角（24～48px），内部卡片 12～16px，按钮 10～14px，**全项目≤3 档**。
- 可选纹理：**极低透明度**噪点 / 斜线网屏（5% 以下），避免抢内容。

## 四、信息架构与界面清单（必画 Figma Frame）

建议在 Figma 建 **8 个主 Frame**（375×812 移动 + 800×600 或 16:9 桌面各一版），命名统一前缀 `ZGC-Miner /`。

| # | 界面 | 关键组件 | 备注 |
|---|------|----------|------|
| 1 | 启动页 `START` | Logo 区、副标题、主 CTA、官网链接区 | 背景可用「学院剪影 + 神经网络线稿」扁平插画 |
| 2 | 关卡导语 `INTRO` | 阶段标签、挑战名、引用式描述、进入任务 | 每关可换一张小 icon |
| 3 | 教程遮罩 `TUTORIAL_OVERLAY` | 半透明罩层、规则列表、底部主按钮 | 与游戏画布同尺寸 |
| 4 | 战斗中 `PLAYING` | 顶栏（积分/达标线/时间/阶段）、退出、加成面板、稿件回收按钮 | **与画布像素对齐**的标注 |
| 5 | 达标弹窗 `LEVEL_COMPLETE` | 奖杯、标题、双按钮（补给站 / 跳过） | 第 8 关单独一帧 |
| 6 | 失败弹窗 `GAME_OVER` | 警示 icon、分数展示、重来 | 可与达标弹窗共享组件变体 |
| 7 | 补给站 `SHOP` | 货架区、商品格、价签、店员气泡、下一关 | **木色货架**可升级为矢量木纹 + 金属挂钩 |
| 8 | 毕业 `GRADUATED` | 大标题、累计分、再来一局 | 礼花/粒子可为 Lottie 或序列帧 |

**Figma MCP 使用建议**：在 Figma 自建文件后，把 `figma.com/design/...` 链接交给 Cursor，用 MCP **拉取节点截图与结构**做对照开发；策划案本身不依赖 MCP 生成，MCP 用于**同步设计稿与代码**。

## 五、游戏内 Canvas 与「矢量贴图」资产表

下列建议 **SVG 优先**（Illustrator / Figma 导出 SVG，复杂渐变可 PNG@2x）。尺寸以逻辑画布 **800×600** 为基准导出。

| 资产 ID | 内容 | 风格提示 |
|---------|------|----------|
| `bg-mine-layer` | 矿层背景：渐变 + 稀疏「数据流」曲线 | 无喧宾夺主，对比度低于前景 |
| `bg-surface-line` | 地表分割线（约 y=100） | 可与 HUD 底条融合 |
| `hook-claw` | 钩爪闭合/张开两态（可选） | 线条干净，2～3 色 |
| `rope` | 绳段（可用代码画线 + 贴图端点） | 简化 |
| `avatar-phd` | 顶部博士生剪影（替代或补充 emoji） | Q 版博士帽 + 笔记本 |
| `item-datapoint` | 小数据点 | 像芯片/光点 |
| `item-dataset` | 数据集 | 文件夹 + 小 AI 角标 |
| `item-corpus` | 大语料 | 书堆 / 服务器机柜简化形 |
| `item-rock` | 绊脚石 | 裂纹石 + 小 bug 图标 |
| `item-diamond` | 成果钻石 | 多面体 + 微光（SVG gradient） |
| `item-bag` | 神秘袋 | 福袋 + 「?」 |
| `item-tnt` | 雷 | 经典桶形，可加「维护中」标签玩梗 |
| `item-intern` | 实习生 | 跑步小人简笔 |
| `particle-spark` | 加分飘字旁火星 | 可选 |

**切片规范**：每文件命名 `zgc-{category}-{name}.svg`，统一 24×24 或 48×48 **图标网格**；大尺寸道具按边界框导出。

## 六、玩梗与活泼度的落点（别堆在一处）

1. **补给站**：商品名已是梗主体；可加**店员立绘**一句轮播吐槽（3～5 条 JSON 配置即可）。
2. **加载/过渡**：0.3s 内短句，如「正在同步随机种子…」。
3. **成就式空状态**：本关无 Buff 时，文案可轮换（已实现一条的可扩展为数组随机**固定种子**防刷屏）。
4. **音效**（若后期加）：抓取成功「叮」、石头「沉闷」、钻石「亮」——策划案标注即可，实现另排期。

## 七、与工程对接方式（给前端）

1. **Token**：在 `index.css` 或 Tailwind `theme.extend` 定义与第三节同名变量；组件只引用 token。
2. **静态资源**：`public/assets/ui/`（界面）、`public/assets/sprites/`（Canvas 用 PNG/SVG）。
3. **Canvas**：`drawImage` 加载 `Image()`，集中在一个 `assets.ts` 预加载表，避免散落路径。
4. **响应式**：外框用 CSS 缩放 canvas（现有思路），HUD 用 **与 canvas 同宽的 flex 分区**，设计稿标注 **百分比宽度** 而非纯 px。

## 八、推荐执行顺序（里程碑）

| 阶段 | 产出 | 周期（参考） |
|------|------|----------------|
| M1 | Figma：设计系统 + Frame 1/4/7（启动、战斗 HUD、补给站） | 小团队 3～5 天 |
| M2 | 导出 SVG/PNG + 替换 React 壳层；字体接入 | 与开发并行 3～5 天 |
| M3 | Canvas 道具 8～10 个主贴图 + 背景 | 5～7 天 |
| M4 | 动效（Framer Motion 微调 + 可选 Lottie）与无障碍对比度复查 | 2～3 天 |

## 九、验收标准（简版）

- [ ] 任意界面主文字与背景对比度 ≥ WCAG AA（大标题可适当放宽但需人工看）。
- [ ] 375 宽与 800 宽下**无重叠、无裁切关键按钮**。
- [ ] 所有可点击区域 ≥ 44×44 CSS px（移动端）。
- [ ] 贴图风格统一（线宽、圆角、色板来自同一 Figma 组件库）。

---

## 十、附录：Cursor + Figma MCP 工作流

1. 在 Figma 创建文件，按第四节建 Frame。
2. 将设计组件导出为 **SVG** 放入仓库 `public/assets/`。
3. 把具体节点链接发给开发侧，用 **Figma MCP `get_design_context` / `get_screenshot`** 对齐像素与间距。
4. 代码仍以本项目 **React + Tailwind + Canvas** 为准；Figma 输出为参考，需适配现有组件结构。
