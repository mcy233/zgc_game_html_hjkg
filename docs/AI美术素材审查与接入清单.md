## 适合作参考，不建议整张直接接入

这些图视觉方向很好，但带固定文字、固定数值或完整布局，直接覆盖当前 React 页面会破坏动态内容：

- `主游戏界面UI概念图.png`：适合作整体风格参考，不能直接用作游戏界面背景，因为 HUD 数值、资源位置、按钮都应由代码动态生成。
- `补给站完整页面.png`：视觉风格优秀，但含固定商品、固定积分、固定底部导航；建议后续按它重构 ShopScreen，而不是直接贴整图。
- `开始页.png`：可参考标题区、主按钮和状态按钮设计；当前含占位文字，可考虑重新生成“无文字版本”。
- `阶段介绍页.png`：可参考弹窗结构，但横版比例与当前竖屏 overlay 逻辑不完全一致。
- `毕业结算页.png`：方向可用，但如果要接入，需要生成无文字、可放动态分数的分层素材。
- `成功 or 失败结算界面2.png`：更像组件设计稿，适合作弹窗重构参考。

## 可用但建议重生成更工程友好的版本

- `碎碎念浮窗.png`：气泡质量可用，但每个气泡变体在一张大图里，后续若要替换 Canvas 内实时文字气泡，建议重生成“单个空白气泡，尾巴方向可控，无文字，无装饰过多”的版本。
- `按钮点击反馈.png`：可作为动画参考，但当前锚点图标不符合“放钩子”的准星/钩爪语义；新底部按钮已补足主要交互反馈，后续若要做点击波纹，建议重生成只有蓝色波纹、无固定图标的纯特效帧。
- `背景动态帧：数据网格流动.png` / `背景动态帧：数据网格流动2.png`：画面可用，但不是透明特效，而是带完整网格背景的小画面；如果要叠加到当前背景上，建议重生成透明数据流粒子层。

## 建议补生成

优先补以下素材，后续接入收益最高：

1. **补给商品单图标包**：当前商品图标可从资源合集里临时复用，但 `SEED`、`WIDE_HOOK`、`TIME_BONUS`、`CORPUS_LENS` 仍需要更准确的单独图标。
2. **纯特效帧**：抓取成功粒子、加分浮字背景光、稿件回收爆破、稀有资源光环，最好全部是透明/洋红背景 sprite sheet，不要带固定按钮或固定主体。
3. **ShopScreen 分层组件**：补给站元素已有可用图，但若要完整替换商店界面，建议继续生成横竖屏两套无文字背景框、商品格、说明框和导师气泡，方便代码动态排布。

## 当前接入策略

本轮只接入不会破坏动态逻辑的素材：

- Canvas 背景使用整张背景图，并保留原有暗色覆盖层保证文字与资源可读。
- 场上资源使用切出的 PNG，资源坐标、分数、碰撞、关卡生成逻辑不变。
- 博士角色使用 6 帧待机图，当前按时间轻量轮播。
- 商店与补给图标优先使用切好的图标，缺失项继续回退到 `lucide-react` 图标。

## 手动校准工具

如果自动裁切出现边缘高光被切掉、原图文字残留、阴影边界不干净等问题，优先使用：

```text
docs/手动切图校准工具.html
```

使用方式：

1. 在浏览器中打开该 HTML。
2. 选择 `resources/` 中的原始元素集合 PNG。
3. 拖动红色裁切框，确保高光、阴影和半透明边缘都被完整包含。
4. 调整“洋红容差”和“软边范围”，在深色背景预览下检查是否有紫边。
5. 导出 PNG，或复制右侧 JSON 发回给开发侧按参数批量重切。

如果素材已经切好，但游戏内 UI 模块的位置、大小、层级不准确，优先使用：

```text
docs/UI布局拖动工具.html
```

使用方式：

1. 在浏览器中打开该 HTML。
2. 在左侧选择“主游戏界面”“学院补给站”“通关弹窗”“失败弹窗”等场景预设。
3. 直接拖动元素调整位置，拖动右下角橙色控制点缩放，右侧可精确输入 `x/y/w/h/z`。
4. 需要新增模块时，从左侧“添加模块”中加入按钮、面板、商品卡、底部栏或弹窗框。
5. 调整完成后复制右侧 JSON，开发侧可按这些参数回填到 React/Tailwind 或 Canvas 布局中。

联动工作流：

1. 在 `docs/手动切图校准工具.html` 中切好素材并导出 PNG。
2. 如果只是临时试摆，打开 `docs/UI布局拖动工具.html`，在“切图素材联动”里多选导入刚导出的 PNG。
3. 如果希望布局工具长期引用新素材，把 PNG 保存或覆盖到 `public/assets/...` 下的固定路径，然后在布局工具里“注册本地相对路径”。
4. 切图工具的“复制给 UI 布局工具的素材 JSON”会生成 `layoutAsset`，可粘贴到布局工具右侧 JSON 后点“应用 JSON”，直接加入素材库。

替换游戏当前界面显示效果时，优先走固定路径：

1. 如果只是换图、不改尺寸和用途，把新 PNG 覆盖到原来的 `public/assets/...` 文件名。
2. 在 `docs/UI布局拖动工具.html` 里点击“刷新图片”，或浏览器强制刷新，即可绕过缓存查看新图。
3. 如果换了新文件名，需要在布局工具里改选中元素的“图片路径”，正式接入时还要同步改 `src/App.tsx` 里对应的资源路径映射。
4. 如果游戏运行页没有变化，通常是浏览器缓存或 Vite dev server 缓存，先强制刷新；仍不变再检查游戏代码引用的是不是同一条资源路径。
